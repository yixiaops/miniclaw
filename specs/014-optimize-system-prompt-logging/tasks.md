### T001: 实现 parsePromptSections 辅助函数

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| MVP | ✅ 是 |
| 预估时间 | 15 分钟 |
| 依赖 | 无 |

**描述:**
在 `src/core/agent/index.ts` 中新增 `parsePromptSections` 函数，用于将系统提示词按 `##` 标题解析为章节列表。

**实现要点:**
- 位置：放在 `estimateMessagesTokens` 函数附近（约第 180-230 行区域）
- 输入：完整系统提示词字符串
- 输出：`Array<{ title: string; content: string }>`
- 按 `## ` 开头的行作为章节分隔符
- 第一个 `##` 之前的内容作为 "前言"
- 空输入返回 `[{ title: '前言', content: '(空)' }]`

**验收标准:**
- [ ] 函数正确解析包含多个 `##` 标题的提示词
- [ ] 第一个 `##` 之前的内容被识别为 "前言"
- [ ] 空提示词返回 `[{ title: '前言', content: '(空)' }]`
- [ ] 仅包含标题的提示词，各章节内容显示 "(空)"
- [ ] `###` 三级标题不作为分隔符，归入上一级章节

### T002: 实现 truncateText 辅助函数

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| MVP | ✅ 是 |
| 预估时间 | 10 分钟 |
| 依赖 | 无 |

**描述:**
在 `src/core/agent/index.ts` 中新增 `truncateText` 函数，用于截断超长文本，保留前后部分。

**实现要点:**
- 位置：紧随 `parsePromptSections` 函数之后
- 默认最大长度：400 字符
- 截断策略：前 150 + `... (省略 N 字符) ...` + 后 150
- 空输入返回 "(空)"

**验收标准:**
- [ ] 文本长度 ≤ 400 时返回原文
- [ ] 文本长度 > 400 时返回 "前150 + 省略提示 + 后150"
- [ ] 省略提示中正确显示省略的字符数
- [ ] 空字符串返回 "(空)"
- [ ] 参数校验正确（maxLength ≤ 0 时的处理）

### T003: 修改 logSendContext 方法

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| MVP | ✅ 是 |
| 预估时间 | 15 分钟 |
| 依赖 | T001, T002 |

**描述:**
修改 `src/core/agent/index.ts` 中的 `logSendContext` 方法，将原有的单一打印逻辑替换为分段打印。

**当前实现（第 1034-1036 行）:**
```typescript
const promptPreview = systemPrompt.length > 3000
  ? systemPrompt.substring(0, 3000) + '...'
  : systemPrompt;
this.log(`  ${promptPreview.replace(/\n/g, '\n   ')}`);
```

**替换为:**
```typescript
const sections = parsePromptSections(systemPrompt);
for (const section of sections) {
  this.log(`  [${section.title}] (${section.content.length} 字符):`);
  const truncated = truncateText(section.content);
  this.log(`    ${truncated.replace(/\n/g, '\n    ')}`);
}
```

**验收标准:**
- [ ] 系统提示词按章节结构化打印
- [ ] 每章节显示标题和字符数
- [ ] 超长章节正确截断
- [ ] 日志格式缩进与原有格式一致
- [ ] 不影响其他日志输出
